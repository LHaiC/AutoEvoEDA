# Adapter Layout And Files

## Project Working Directory

Run `evo` from the project repository, or pass `--config path/to/evo.yaml`. Relative paths in `evo.yaml` are interpreted from the config file directory and project repo where the harness runs candidate work.

Recommended project layout:

```text
project-repo/
  evo.yaml
  prompts/
    planner.md
    coder.md
    reviewer.md
    repair.md
    flow.md             # optional domain agent
    mapper.md           # optional domain agent
    logic.md            # optional domain agent
  scripts/
    build.sh
    run_regression.sh
    compare_regression.py
    run_perf.sh
    reward.py
    evo_nonstop_supervisor.sh   # optional local long-run launcher
  results/              # generated by scripts, usually forbidden edits
  .evo/                 # generated harness state, do not hand-edit except comments/state controls
```

## Required Adapter Inputs

- `evo.yaml`: schema, repo path, champion branch, worktree root, guards, pipeline commands, result files, memory, agents, and optional domain agents.
- `prompts/coder.md`: optimization goal, correctness constraints, allowed implementation scope, forbidden edits, and small-patch rule.
- `scripts/build.sh`: exits 0 only when the candidate builds.
- `scripts/run_regression.sh`: exits 0 only when correctness/regression/equivalence passes.
- `scripts/compare_regression.py`: optional separate comparator; exits non-zero on mismatch.
- `scripts/run_perf.sh`: produces performance or QoR result files.
- `scripts/reward.py`: exits 0 only when the reward/acceptance target passes.

## Generated State

- `.evo/runs/<run_id>/`: per-candidate phase docs, patch, events, evaluator results, checkpoint state, reproducibility info.
- `.evo/events.jsonl`: repository-level event log.
- `.evo/history.jsonl`: decisions and promotion history.
- `.evo/brief.md`: concise shared history for quick agent lookup.
- `.evo/session/`: daemon status, active run marker, comments, pause/resume state.
- `.evo/agents/<session_id>/`: framework-level role memory, transcript, last prompt, last response, per-call exchange files.
- `.evo/agents/interactions.jsonl`: explicit planner/coder/reviewer/repair/understanding interaction chain.
- `.evo/runs/<run_id>/agent_flow.md`: per-run agent sequence with links to prompts and responses.
- `.evo/memory/`: project memory, lessons, agent handoffs, rejected ideas, accepted patterns, rulebase, code-understanding memory.

## Worktree Layout

Set `workspace.worktree_root` outside the project repo when possible, for example `../.evo-worktrees`. Candidate worktrees are preserved for inspection until removed with `evo worktree cleanup`.
